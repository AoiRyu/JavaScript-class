The Document Object Model (DOM)

What is the DOM?

DOM defines the logical structure of the page

The HTML DOM is a standerd for how to get, change, add, or delete HTML elements

Syntax test:
document.body.innerHTML = 'Some text';

You star with this property and you can be used to get or change any HTML element

DOM structure is:
document ~> html ~> head/body
head ~> title
body ~> h1/a

As you can see, the DOM describes the relationships of HTML elementes (nodes). Element objects can have child nodes. Elements on the same node tree level are called siblings

In the example above tag H1 and A are siblings. TITLE is a child of HEAD, and so on

Node Relationship

DOM represents a document as a tree which is made up of parent-child relationships. A parent can have one or many child nodes.

The basic HTML structure.
<html>
<head>
<title> TITULO </title>
</head>
<body>
THE BODY
</body>
</html>

The text below demonstrates the relationship between the nodes.

Root elemente <html> => last child ~> Element <body> and first child ~> Element <head>
Elemente <head> ~> parentNode ~> <html> and ~> nextSibling ~><body>
Elemente <body> ~> previsiousSibling ~><head>

well, let's understand better:

  • html - the root node, has 2 children (head and body) and no parents;
  • head - the first child of html, has one child title, and one parent html;
  • title - has one parent head;
  • body - is the last child of html, and has no children
